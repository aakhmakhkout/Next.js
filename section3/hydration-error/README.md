# Hydration Errors

## What is a Hydration Error?

A Hydration Error happens when the **HTML generated by the server** is different from what **React expects when it runs in the browser**.

During hydration, React compares

```
Server HTML

↓

Browser (Virtual DOM)
```

If both don't match, React throws a hydration error.

---

## Why do Hydration Errors happen?

Hydration errors happen whenever the server and browser render **different content** for the same component.

Think of it like this

```
Server

↓

<h1>Hello</h1>

Browser expects

↓

<h1>Hi</h1>

↓

Hydration Error
```

The browser already has one HTML structure, but React expects another.

---

## Common Causes

### 1. Browser APIs

Example

```jsx
if (typeof window === "undefined") {
    return <h1>Server</h1>;
}

return <h1>Browser</h1>;
```

### Server

```
<h1>Server</h1>
```

### Browser

```
<h1>Browser</h1>
```

Both outputs are different.

↓

Hydration Error.

---

### 2. Date & Time

Example

```jsx
<h1>{Date.now()}</h1>
```

Server renders

```
1721456231
```

A few milliseconds later

Browser renders

```
1721456239
```

Values don't match.

↓

Hydration Error.

---

### 3. Math.random()

Example

```jsx
<h1>{Math.random()}</h1>
```

Server

```
0.4521
```

Browser

```
0.9174
```

Different values.

↓

Hydration Error.

---

### 4. Browser Extensions

Sometimes your code is completely correct.

But extensions like

- Dark Reader
- Grammarly
- Google Translate
- Password Managers

modify the browser DOM before React hydrates.

Now React compares

```
Expected DOM

↓

Modified DOM
```

Since both are different,

Hydration Error occurs.

---

## My Biggest Confusion

I thought

> React compares the server file with the client file.

Not true.

React compares

```
HTML already present in Browser

↓

Virtual DOM created by React
```

The source files are never compared.

Only the rendered output is.

---

## Development vs Production

### Development

Hydration mismatches are shown as **errors** in the browser.

React displays a detailed message explaining

- Which component caused the problem.
- Which HTML didn't match.
- Possible reasons.

This helps developers fix the issue.

---

### Production

Users usually **don't see the hydration error message**.

Instead,

React tries to recover automatically.

Flow

```
Server HTML

↓

Hydration fails

↓

React discards the mismatched part

↓

Client renders it again

↓

User sees the correct UI
```

So in production, the page usually still works, but React has to do extra work because hydration failed.

---

## How to Avoid Hydration Errors

Avoid rendering different content on the server and browser.

Common practices

- Don't use `Date.now()` directly during rendering.
- Don't use `Math.random()` during rendering.
- Avoid checking browser APIs (`window`, `document`, `localStorage`) while rendering.
- Use `useEffect()` for browser-only code.
- Keep the initial server and client render identical.

## Easy Memory Trick

Hydration succeeds

```
Server HTML

==

Browser Virtual DOM
```

Hydration fails

```
Server HTML

≠

Browser Virtual DOM
```

Simple as that.

---

## Final Understanding

A Hydration Error occurs when React finds that the HTML already present in the browser does not match the Virtual DOM it generates during hydration. This commonly happens when the server and browser render different values, such as timestamps, random numbers, browser-specific APIs, or content modified by browser extensions. During development, React displays detailed hydration errors to help developers identify the mismatch. In production, React usually recovers by discarding the mismatched section and rendering it again on the client, allowing the application to continue functioning without exposing the error to end users.
